Setting up my own Cloud server with Nextcloud and Raspberry Pi 4

I think that this was the most exciting project for me so far. It all started when I was looking for an alternative cloud for my non profit organization. Google Drive was not secure enough for sensitive data and Dropbox has too little storage. Since I always wanted to do a project with a Raspberry Pi, I looked for inspiration on the internet. At this point I would like to thank my parents for giving me one for Christmas.

To do this, I first downloaded the NextCloudPi image, flashed it and transferred it to a MicroSD card. This is inserted into the SD card port of the Pi and the small mini-computer is started. A monitor is recommended for the initial setup. This way you can follow what your NextCloudPi is doing. You will first see that the storage space is expanding and the Pi is then restarted. You will also see which IP address the Raspberry Pi has received from the DHCP server. You don't need the screen if the Pi is connected via Ethernet cable and you can read out in the router or in the DHCP server which IP address the Pi has received. Then the setup is completely headless and that's really great. However, you should also know that the NextCloudPi can be reached via nextcloudpi.local if your setup is configured with it. This is very practical, but does not necessarily have to work.

If you know the IP address of your Pi, enter it in the address bar of your browser. The browser will complain that it is an insecure certificate. This is because the certificate itself is signed and you can ignore the warning and still be redirected to the corresponding website. In order to access the system with the excellent Nextcloud app from the Internet, a DynDNS domain is required. But what is a DynDNS domain anyway?

In most cases, the internet provider regularly causes a forced separation in order to make the possible operation of, for example, a company server on a DSL private connection more difficult. For this reason, you need a dynamic DNS domain, which is configured in the router. As soon as the router is assigned a new IP address by the provider, it sends this to the DynDNS service and thus updates the stored data. This makes it possible to enter one's own internet address, such as nextcloud.example.com, and be automatically redirected to the correct IP of one's own router. Finally, a port redirection within the router is required. Ports 80 and 443 should be redirected internally to the RaspberryPi address. If everything has been configured correctly, it should be possible to access the Miniserver from outside via a smartphone app.

Data Visualization with Tableau

I wanted to make good use of the time during the lockdown and have been learning Tableau. After the program unfortunately came a bit short in our university, I decided to learn it with Youtube. First, I downloaded the official data on corona infections from the WHO. In the next step, I visualized the top 10 countries with the highest death and infection rates in a dashboard. Unfortunately, the online embedding is not the best (I think this is due to Tableau public), which is why the play button works but doesn't speed up. Anyway, feel free to download and use the dashboard.
